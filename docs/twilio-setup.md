# Twilio Console Setup

You need a Twilio account, a phone number, and (for production SMS in the US) a
Messaging Service registered for A2P/10DLC.

## 1. Credentials

Copy your Account SID and Auth Token from the Twilio console into `.env`:

```env
TWILIO_ACCOUNT_SID=ACxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
TWILIO_AUTH_TOKEN=your-auth-token
```

## 2. Public webhook base URL

Twilio must reach your app over public HTTPS. Set the public URL so signature
validation stays correct behind a TLS-terminating proxy:

```env
PHONE_WEBHOOK_BASE_URL=https://your-app.example.com
TWILIO_VALIDATE_WEBHOOKS=true
```

Locally, expose your app with a tunnel (`cloudflared tunnel`, `ngrok http`) and
set `PHONE_WEBHOOK_BASE_URL` to the tunnel URL.

## 3. Configure the number's webhooks

In **Phone Numbers → Manage → Active numbers → (your number)**, set:

| Field | URL (method POST) |
| --- | --- |
| Messaging → A message comes in | `https://your-app.example.com/phone/twilio/sms/inbound` |
| Voice → A call comes in | `https://your-app.example.com/phone/twilio/voice/inbound` |

The remaining callbacks (SMS status, dial status, call status, recording,
transcription, AI status) are generated by the package automatically and do not
need to be set in the console. Routes are listed in
[Webhook security](webhooks-security.md).

## 4. Sender for outbound SMS

Prefer a Messaging Service (manages the 10DLC campaign, sender pool, and
sticky-sender behavior):

```env
TWILIO_MESSAGING_SERVICE_SID=MGxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

Or use a plain from-number:

```env
TWILIO_FROM=+16615550100
```

## 5. Verify

```bash
php artisan phone:doctor          # offline config check
php artisan phone:doctor --live   # also pings Twilio with your credentials
```

See [Compliance](compliance.md) for 10DLC registration and consent obligations.
